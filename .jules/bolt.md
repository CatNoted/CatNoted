## 2024-07-24 - Document Editor Performance Analysis\n**Learning:** Found potential performance optimization. DocumentEditor renders blocks by iterating over blocks.map without memoization on the individual blocks (TextBlock, HeadingBlock, etc.). This causes all blocks to re-render whenever ANY block is modified (as it triggers a new blocks array from the useDocumentStore hook).\n**Action:** Use React.memo on block components (TextBlock, HeadingBlock) to prevent unnecessary re-renders when other blocks are updated.
## 2024-07-24 - Canvas Card Performance Analysis
**Learning:** Found potential performance optimization in Canvas. `CanvasCard` does not use `React.memo`, meaning all cards re-render whenever the infinite canvas state updates (e.g., selection changes, panning, zooming). This could be a bottleneck on large canvases.
**Action:** Use `React.memo` on `CanvasCard` (and maybe other block/canvas components) to prevent unnecessary re-renders when other canvas elements or states are updated.

## 2024-07-24 - React Component Memoization Optimization
**Learning:** When iterating over an array of items to render React components (e.g., `blocks.map`), if the parent component's state changes (like when one block updates), *all* mapped items will re-render if not memoized. However, simply using `React.memo` on the child component is not enough if the parent passes down inline functions or newly created objects as props (e.g., `onChange={(val) => ...}`).
**Action:** To properly prevent unnecessary re-renders in a list, extract the mapped item into its own component (`BlockRow`), use `React.useCallback` inside this new component to stabilize the function references, and pass these stable references to the memoized child components (`TextBlock`, `HeadingBlock`, etc.).
