# AI Space Agent Runtime & Sandbox — Agent Workspace Guidelines

This package (`packages/agent-runtime`) coordinates sandboxed code execution, client-side files, and LLM orchestration.

---

## 1. Responsibilities

- **IndexedDB VFS**: Keeps user document structures, settings, and skills cached locally in a file system layer.
- **Isolated IFrame Sandbox**: Spawns an iframe with `sandbox="allow-scripts"` to securely execute dynamic HTML/JS/CSS widgets generated by the AI agent.
- **BYOK LLM Request Client**: Communicates directly with LLM API endpoints (Gemini, OpenAI, Anthropic, Ollama) using user-supplied local keys.

---

## 2. Technical Stack & Constraints

- **Communication Protocol**: strict `window.postMessage` serialization using types from `@catnoted/shared`.
- **CSS Variable Injection**: Automatically copies main thread theme variables (CSS variables) into the sandbox’s Shadow DOM or inline style sheets for light/dark mode compliance.
- **Security Check**: Do not allow widgets access to root documents (`window.parent` / `window.top`) or cookies.
